NAMING LOGIC

HappyPause Image Naming Convention Specification
1. Purpose
Each activity image filename encodes a unique 12-digit activity ID.
This ID must match the ID stored in the database for that activity.
The filename structure ensures:
•	Unique identification
•	Category grouping
•	Future scalability
•	Compatibility with database logic
________________________________________
2. File Naming Structure
[CATEGORY]-[12_DIGIT_ID].png
Example:
FITNESS-000100000001.png
________________________________________
3. 12-Digit ID Structure
The 12-digit ID is structured as follows:
[CUSTOM_FLAG][CATEGORY_CODE][ACTIVITY_SEQUENCE]
Segment	Length	Description
CUSTOM_FLAG	1 digit	0 = default activity / 1 = custom user-created
CATEGORY_CODE	3 digits	Encoded category
ACTIVITY_SEQUENCE	8 digits	Sequential number within category
________________________________________
4. Category Codes
Category	Code
FITNESS	001
LEISURE	002
SOCIAL	003
MIND	004
SPIRITUAL	005
RELAXATION	006
________________________________________
5. Example Breakdown
Filename:
FITNESS-000100000001.png
Breakdown:
0 001 00000001
│ │   └────── Activity #1
│ └────────── FITNESS category
└──────────── Default (not custom)
Meaning:
•	Default activity
•	Category: FITNESS
•	First activity in that category
________________________________________
6. App Usage Logic
Extracting ID
When loading the activity:
1.	Read the filename.
2.	Extract the 12-digit ID.
3.	Match it to the activity record in the database.
4.	Display image where activity.id == extracted_id.
________________________________________
Determining Category from ID
In code:
category_code = id[1:4]
Then map:
•	001 → FITNESS
•	002 → LEISURE
•	etc.
________________________________________
Determining Custom vs Default
isCustom = id[0] == "1"
________________________________________
7. Why This Structure Matters
This system:
•	Avoids duplicate images
•	Allows infinite scaling (up to 99,999,999 activities per category)
•	Separates system content from user-generated content
•	Makes the filename itself a lightweight database key
________________________________________
8. Important Rule
The filename ID MUST always match the database ID.
The app should treat the 12-digit number as the single source of truth.
